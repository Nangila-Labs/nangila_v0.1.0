# Nangila v0.1 Support Matrix

This document defines the explicit support contract for `v0.1`.

## Supported Product Surface

`v0.1` supports:
- Python package installation from source with `maturin`
- Rust core compression logic
- PyTorch `DistributedDataParallel` integration
- Calibration with `Sculptor`
- Mask-backed hook creation with `NangilaHook`
- Default non-CUDA builds

`v0.1` does not support:
- FSDP as a stable feature
- low-level CUDA helper APIs as a stable feature
- multi-node qualification
- production guarantees for DGC/PowerSGD-specific tuning paths
- broad HPC or digital-twin workloads

## Environment Matrix

### Stable

| Area | Support |
|------|---------|
| OS | Linux |
| macOS | Development and smoke-test only |
| Python | 3.10 - 3.12 |
| PyTorch | 2.x, validated through source install smoke path |
| Build path | `maturin develop --release -F python` |
| Distributed mode | `DistributedDataParallel` |
| CUDA | Optional, not part of stable support |

### Experimental

| Area | Status |
|------|--------|
| FSDP | Experimental |
| Low-level CUDA Python helpers | Experimental |
| GPU-native FSDP hook | Experimental |
| Multi-node | Experimental |
| Performance claims beyond smoke validation | Experimental |

## GPU Validation Notes

- A 1-GPU CUDA smoke pass is useful for build/import/runtime pre-qualification.
- The single-GPU smoke lane does not qualify `v0.1` distributed correctness.
- The `v0.1` DDP qualification gate is a 2-GPU parity/stability run on one supported Linux + CUDA setup.
- That gate has been exercised successfully on a 2x NVIDIA GPU Linux host as part of the `v0.1.0` release pass.

## Test Contract

The default green baseline for `v0.1` is:
- `cargo test`
- `python -m pytest -q` after editable install from a local virtual environment

CUDA-specific release qualification is:
- `bash scripts/cuda_single_gpu_smoke.sh` for one-GPU build/import/runtime pre-qualification
- `CUDA_VISIBLE_DEVICES=0,1 python -m torch.distributed.run --standalone --nproc_per_node=2 tests/integration/test_ddp_correctness.py` for the required two-GPU DDP qualification gate

Distributed and GPU-heavy tests are opt-in via `NANGILA_RUN_INTEGRATION=1`.
