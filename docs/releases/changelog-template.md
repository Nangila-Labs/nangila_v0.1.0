# Changelog Template

Copy this structure into `CHANGELOG.md` for the next release entry.

```md
## vX.Y.Z - YYYY-MM-DD

### Added
- User-visible addition

### Changed
- User-visible behavior change

### Fixed
- Bug fix with user impact

### Support
- Support-matrix change
- Compatibility note

### Experimental
- Experimental feature update, if relevant
```

Guidelines:
- Keep entries user-facing.
- Do not list internal refactors unless they affect release behavior.
- Separate stable support changes from experimental changes.
