# Docs Layout

The documentation tree is split by purpose.

- `releases/`: active support contracts, release criteria, templates, and release-process material
- `strategy/`: product, licensing, and longer-range roadmap material
- `archive/`: historical reports, whitepapers, reference material, and archived logs not required for the active release contract

- [`releases/v0.1-release.md`](/Users/craigchirara/nangila/docs/releases/v0.1-release.md)
